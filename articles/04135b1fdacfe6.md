## はじめに

今年（2025年）の2月に自作PCを組んで運用していたのですが、10月の中旬頃から起動が不安定になっていました。

結論から言うと、`Ryzen 9800X3D`（CPU）の故障が原因で起こった不具合でした。

`9800X3D`を購入した当時は品薄で国内ではなかなか手に入らなかったため`newegg`から個人輸入で購入しており、色々調べていると並行輸入品は保証の対象外だとか何だとかを見たので正直終わったと思いました。

最終的にAMDのRMA（返品・交換）対応で解決したのでその過程を共有できたらなと思います。

きっと私と同じように`9800X3D`の不具合で悩んでいる方や個人輸入で諦めそうになっている方は、きっと役に立つと思いますので、ぜひ参考にしてください。

## PC構成

- **CPU**: AMD Ryzen 7 9800X3D
- **マザーボード**: ASRock B650 Steel Legend WiFi
- **メモリ**: Corsair DDR5-6000 32GB (16GB×2) CMH32GX5M2E6000C36W
- **GPU**: AMD Radeon RX 7900 XTX TAICHI
- **SSD**: WD BLACK SN850X
- **電源**: Corsair RM1000x Shift

## 発生した症状

### 初期症状

初めて出た症状は、夜ご飯を食べるためにPCを放置しスリープ状態へ。その後スリープから復帰させようとしたところ、ケースファンやグラボのライトは付くものの画面がつかず、マザーボードを見てみると`boot`ランプが点灯していました。

その日は電源ボタンを長押しで電源OFF→電源投入で起動しました。

### 不安定に

翌日、シャットダウンの状態から電源を投入したところ、同じくマザーボードの`boot`ランプが点灯しWindowsが起動しませんでした。

電源を落とし再度電源を投入をしてみますが電源がつきません。3回ほど繰り返しWindowsの起動に成功しました。

### 症状の悪化

相変わらずスリープからの復帰はできず、さらに電源OFFからの起動も10回ほど繰り返さないと起動しないようになりました。

さすがに何かがおかしいと思ったので色々試してみることに。

## 実施したトラブルシューティング

### 1: BIOS設定の調整

メモリのオーバークロックを利用していたため、メモリ周りのBios設定をデフォルトにしてみました。

```
✅ EXPO/XMPプロファイル無効化
✅ メモリクロックをDDR5-4800（JEDEC標準）に設定
✅ メモリ電圧の手動調整（1.25V〜1.40V）
✅ SOC電圧の調整（1.20V〜1.30V）
✅ Infinity Fabric周波数の変更（1200MHz〜2000MHz）
✅ CMOSクリア（複数回実施）
✅ 全設定をOptimized Defaultsに戻す
```

**結果**: 改善せず

### 2: Windowsのクリーンインストール

`boot`ランプが点灯しているのはWindowsのブートシステムがおかしくなっている可能性があるという情報を見たのでクリーンインストールしました。

また、biosが組んだときのままだったので念の為最新化しました。

```
✅ Windowsのクリーンインストール
✅ BIOSを最新版に更新
```

**結果**: 改善せず

### 3: ハードウェアの物理確認

ソフトウェアの問題がない事を確認したので、ハードウェアを順に確認しました。

```
✅ 全電源ケーブル（24pin、CPU 8pin、GPU）の再接続
✅ SSDを取り外して起動テスト
✅ GPUの再装着・補助電源ケーブルの確認
✅ GPUなしで起動（9800X3Dの内部GPUを使用）
✅ メモリの再装着
✅ メモリ1枚構成でテスト（両方のメモリ、全スロットで試行）
✅ CPUの再装着と接触確認
```

**結果**: 改善せず

### 4: ハードウェアの診断

診断ソフトを使ってハードを確認しました。

```
✅ CrystalDiskInfoを利用しSSDを診断
✅ Windowsメモリ診断ツールを利用しメモリを診断
✅ Memtest86を利用しメモリを診断
```

**結果**: 問題なし

### 5: ハードウェアの変更

新たにメモリ、マザーボードを新調して交換してみました。

```
✅ マザーボードを別製品に交換
   - ASRock B650 Steel Legend WiFi
     ↓
   - Gigabyte B850 AORUS STEALTH ICE
✅ メモリを別製品に交換
   - Corsair DDR5-6000 32GB (16GB×2) CMH32GX5M2E6000C36W
     ↓
   - Corsair DDR5-5600 64GB (32GB×2) CMK64GX5M2B5600C40W
```

**結果**: 改善せず

## RMAの申請

AMDの保証サービスよりRMA申請を行い無事新しいCPUを手に入れました。
申請後からおおよそ11日で新しいCPUが手元に届きました。

組み付け後2日経たずでこの記事を執筆していますが、今のところ起動不良はありません。

## まとめ

### 症状のチェックリスト

以下の症状がある場合、CPU不良の可能性があるため保証を使って交換してもらうのが良いかもしれません。
```
☑ 複数回の再起動が必要（2回以上）
☑ JEDEC標準速度でも不安定
☑ スリープから復帰しない
☑ 異なるマザーボードでも同じ症状
☑ POSTコードでCPU関連エラー
☑ エラーランプが不規則に変化
```

### トラブルシューティングの順序

```
1. BIOS設定をデフォルトに戻す
   └→ 改善なし
2. メモリをJEDEC標準速度（DDR5-4800）に設定
   └→ 改善なし
3. メモリ1枚構成でテスト
   └→ 改善なし
4. 可能ならメモリ、マザーボードを別製品でテスト
   └→ 改善なし
5. CPU不良と判断 → RMA申請
```

### RMA手続きのスケジュール

```
Day 1-3: AMDサポートとのやり取り
Day 4: RMA承認
Day 4: 不良品発送（DHL）
Day 5-7: AMD到着・検査完了
Day 8: 交換品発送
Day 9-11: 交換品到着

合計: 11日
```

### 費用
- DHL発送費: 約13,000円
- 交換品の返送: AMD負担（無料）

## 最後に

個人輸入品でも保証を受けれて本当に良かったです。もしこれが受け入れられなかったらまた7万円もかかるのかと恐ろしいばかりです。

とはいえ送料で13,000円かかっているんですけど…

これからは多少高くとも、多少品薄だったとしても国内正規品を購入しようと思いました。

英語対応にはなりますがAMDのサポートは丁寧で、事細かに状況を説明すると迅速に対応してくれました。

結局なぜ故障したのかはわかりませんが、やっぱりAsrockのマザーボードで使ってたからなのでしょうかね。もし皆さんのPCも9800X3D + Asrockマザボで構成されていたら注意してください。

RMA申請の詳しい方法は別記事「[AMD Ryzen 7 9800X3D RMAの方法 - 申請から交換品到着まで](https://zenn.dev/tsuruya/articles/04135b1fdacfe6)」で解説していますので、申請を考えている方はそちらをご覧ください。
